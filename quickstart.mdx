<Note>
  rshell is **default-deny**: no commands, file access, or environment variables are available unless you explicitly permit them. This is by design — it prevents AI agents from accidentally or maliciously escaping their sandbox.
</Note>

## Prerequisites

- Go 1.25 or later (rshell requires Go 1.25+; uses `os.Root` for filesystem sandboxing)
- A Go module project (`go mod init`)

## Get started

<Steps>
  <Step title="Install rshell">
    Add rshell to your Go module:

    ```bash
    go get github.com/DataDog/rshell
    ```

    rshell depends on [`mvdan.cc/sh/v3`](https://pkg.go.dev/mvdan.cc/sh/v3) for shell parsing. It is pulled in automatically.
  </Step>

  <Step title="Parse a shell script">
    rshell uses `mvdan.cc/sh/v3/syntax` to parse shell scripts into an AST before execution:

    ```go main.go
    import (
        "strings"
        "mvdan.cc/sh/v3/syntax"
    )

    script := `echo "hello from rshell"`

    prog, err := syntax.NewParser().Parse(strings.NewReader(script), "")
    if err != nil {
        // Syntax errors return exit code 2
        log.Fatal(err)
    }
    ```

    The parsed `prog` is a `*syntax.File` that can be passed to `runner.Run()`.
  </Step>

  <Step title="Create a Runner with security options">
    Use `interp.New()` to create a sandboxed runner. Configure exactly what the script is allowed to do:

    ```go main.go
    import (
        "os"
        "time"
        "github.com/DataDog/rshell/interp"
    )

    runner, err := interp.New(
        interp.StdIO(nil, os.Stdout, os.Stderr),
        interp.AllowedCommands([]string{"rshell:echo"}),
        interp.MaxExecutionTime(5*time.Second),
    )
    if err != nil {
        log.Fatal(err)
    }
    defer runner.Close()
    ```

    <Tip>
      Always `defer runner.Close()` immediately after `interp.New()`. Close releases the `os.Root` file descriptors held by `AllowedPaths`. It is safe to call multiple times.
    </Tip>
  </Step>

  <Step title="Run the script">
    Pass the parsed AST and a context to `runner.Run()`:

    ```go main.go
    import "context"

    err = runner.Run(context.Background(), prog)
    if err != nil {
        log.Printf("script error: %v", err)
    }
    ```

    `Run()` is safe to call multiple times on the same runner to interpret scripts incrementally. Call `runner.Reset()` between runs if you want to clear shell state (variables, working directory).
  </Step>

  <Step title="Handle exit codes">
    `runner.Run()` returns nil on success, or an error on failure. Use `errors.As` to extract the exit code:

    ```go main.go
    import (
        "errors"
        "github.com/DataDog/rshell/interp"
    )

    err = runner.Run(context.Background(), prog)

    var status interp.ExitStatus
    switch {
    case err == nil:
        fmt.Println("success")
    case errors.As(err, &status):
        fmt.Printf("exited with code %d\n", int(status))
    default:
        fmt.Printf("fatal error: %v\n", err)
    }
    ```

    | Exit code | Meaning |
    |-----------|---------|
    | 0 | Success |
    | 1 | Builtin or command error |
    | 2 | Blocked shell feature or syntax error |
    | 127 | Command not allowed (not in AllowedCommands) |
  </Step>
</Steps>

## Complete example

Here is the full working example from the README:

```go main.go
package main

import (
    "context"
    "os"
    "strings"
    "time"

    "github.com/DataDog/rshell/interp"
    "mvdan.cc/sh/v3/syntax"
)

func main() {
    script := `echo "hello from rshell"`

    prog, _ := syntax.NewParser().Parse(strings.NewReader(script), "")

    runner, _ := interp.New(
        interp.StdIO(nil, os.Stdout, os.Stderr),
        interp.AllowedCommands([]string{"rshell:echo"}),
        interp.MaxExecutionTime(5*time.Second),
    )
    defer runner.Close()

    runner.Run(context.Background(), prog)
}
```

## Using the CLI

The `rshell` binary provides a command-line interface for running shell scripts directly:

```bash
# Install the CLI
go install github.com/DataDog/rshell/cmd/rshell@latest

# Run an inline command (allow all builtins, 5s timeout)
rshell --allow-all-commands --timeout 5s -c 'echo "hello from rshell"'

# Run with specific allowed commands and restricted paths
rshell --allowed-commands rshell:cat,rshell:grep \
       --allowed-paths /var/log \
       -c 'grep "ERROR" /var/log/app.log'

# Run a shell script file
rshell --allowed-commands rshell:echo,rshell:ls script.sh
```

### CLI flags

| Flag | Short | Description |
|------|-------|-------------|
| `-c <cmd>` | `-c` | Shell command string to execute |
| `--allowed-commands` | | Comma-separated namespaced commands (e.g. `rshell:cat,rshell:find`) |
| `--allowed-paths` | `-p` | Comma-separated list of allowed directories |
| `--allow-all-commands` | | Allow all commands — for testing only |
| `--timeout` | | Max execution time (e.g. `100ms`, `5s`, `1m`) |
| `--proc-path` | | Path to proc filesystem for `ps` builtin (Linux only) |

<Warning>
  `--command` (the long form) is intentionally rejected. Only `-c` is supported, matching bash convention.
</Warning>
